USER STORY: Identifier Swap (IMEI/Serienummer Wijziging)

Als magazijnmedewerker
Wil ik de IMEI of serienummer van een apparaat wijzigen
Zodat ik registratiefouten kan corrigeren zonder tracking geschiedenis te verliezen

ACCEPTATIECRITERIA:

IDENTIFIER SWAP FUNCTIONALITEIT:
- [ ] "Identifier Wijzigen" knop op identifier detailpagina
- [ ] Modal dialoog met formulier velden:
  - Huidige IMEI (alleen-lezen, indien van toepassing)
  - Nieuwe IMEI (bewerkbaar, als tracking modus vereist)
  - Huidig Serienummer (alleen-lezen, indien van toepassing)
  - Nieuw Serienummer (bewerkbaar, als tracking modus vereist)
  - Optioneel notitie veld voor reden

VALIDATIE REGELS:
- [ ] Nieuwe waarde kan niet identiek zijn aan huidige waarde
- [ ] Nieuwe IMEI/Serienummer moet uniek zijn over alle identifiers
- [ ] Toon alleen velden relevant voor productversie tracking modus:
  - IMEI tracking: Alleen IMEI velden
  - Serienummer tracking: Alleen Serienummer velden  
  - Beide tracking: Beide veld sets
- [ ] Minimaal één veld moet gewijzigd worden
- [ ] Vereiste velden gebaseerd op tracking configuratie

SWAP UITVOERING:
- [ ] Werk device_identifiers tabel bij met nieuwe waarden
- [ ] Creëer entry in identifier_history tabel:
  - device_identifier_id
  - old_imei, new_imei
  - old_serial_number, new_serial_number
  - changed_by (huidige gebruiker)
  - note (optionele reden)
  - timestamp

GESCHIEDENIS WEERGAVE:
- [ ] Identifier Geschiedenis sectie op detailpagina
- [ ] Tabel met alle identifier swaps:
  - Datum/Tijd
  - Oude IMEI → Nieuwe IMEI (indien gewijzigd)
  - Oud Serienummer → Nieuw Serienummer (indien gewijzigd)
  - Gewijzigd Door
  - Reden/Notitie
- [ ] Duidelijke visuele indicatie van wat gewijzigd is

BEDRIJFSREGELS:
- [ ] Kan niet swappen naar identifiers die al bestaan
- [ ] Alle swaps worden permanent gelogd
- [ ] Productversie en status blijven ongewijzigd
- [ ] Gebruiker moet ingelogd zijn om swaps uit te voeren
- [ ] Bevestiging vereist voor uitvoering swap

UI VEREISTEN:
- [ ] Professionele modal dialoog
- [ ] Duidelijke veld labels die oud vs nieuwe waarden tonen
- [ ] Formulier validatie met behulpzame foutmeldingen
- [ ] Bevestigingsdialoog met samenvatting van wijzigingen
- [ ] Succesbericht na voltooiing
- [ ] Bijgewerkte detailpagina met nieuwe waarden en geschiedenis
